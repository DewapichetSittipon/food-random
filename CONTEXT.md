# FridgeChef

แอป PWA แบบ Offline-First สำหรับสุ่มเมนูอาหารไทยจากวัตถุดิบที่ผู้ใช้มีในตู้เย็น — matching และ random ทำงานในเครื่องทั้งหมด

## Language

**Recipe (เมนู)**:
หนึ่งจานที่สุ่มได้ ประกอบด้วยวัตถุดิบหลัก เครื่องปรุง ขั้นตอนทำ และข้อมูลประกอบ (เวลา/ความยาก/จำนวนที่) — ใน UI ภาษาไทยแสดงคำว่า "เมนู" ได้ แต่ในโค้ดและ DB ใช้ Recipe เสมอ
_Avoid_: Menu, Dish, Food

**Ingredient (วัตถุดิบ)**:
สิ่งที่ผู้ใช้เลือกว่ามีในตู้เย็น และเมนูใช้อ้างอิงเพื่อเช็กว่าทำได้จริงไหม มีตัวตนถาวรเป็น slug id (เช่น `pork-minced`) คู่กับชื่อไทยสำหรับแสดงผล — "หมู" กับ "หมูสับ" เป็นคนละ Ingredient
_Avoid_: item, ของ, เครื่องปรุง

**Seasoning (เครื่องปรุง)**:
ของปรุงรสที่ถือว่าครัวมีอยู่แล้ว ผูกกับเมนูพร้อมปริมาณ (เช่น น้ำปลา 1 ชช.) แสดงในหน้าวิธีทำเท่านั้น **ไม่นำมาเช็กตอน matching**
_Avoid_: ingredient, ส่วนผสม

**Core Ingredient (วัตถุดิบหลัก)**:
Ingredient ที่ผูกกับ Recipe — คือเส้นความสัมพันธ์ Recipe↔Ingredient หนึ่งเส้น ผู้ใช้ต้องมีครบทุกตัว Recipe จึง Cookable (ยกเว้นตัวที่เป็น Protein ซึ่งมีทางเลือกใดทางเลือกหนึ่งก็พอ)

**Required Ingredient (วัตถุดิบบังคับ)**:
Core Ingredient ที่ติด flag ว่าขาดไม่ได้และแทนกันไม่ได้ (เช่น ใบกะเพราของผัดกะเพรา, ไข่ของไข่เจียว)
_Avoid_: requiredTags (ชื่อเดิมในสเปก — มันไม่ใช่ tag)

**Protein (โปรตีน)**:
Core Ingredient ที่ติด flag ว่าเป็นเนื้อสัตว์/โปรตีนหลักของจาน เมนูที่มี Protein หลายตัวคือ "ทางเลือก" — ผู้ใช้มีตัวใดตัวหนึ่งก็ Cookable (เช่น ไข่เจียวหมูสับ ใช้ไข่+หมูสับ หรือไข่ล้วนก็ได้) แต่ไม่มีเลยถือว่าทำไม่ได้ — Ingredient ตัวเดียวเป็นทั้ง Required และ Protein ได้ (เช่น กุ้งในต้มยำกุ้ง)

**Cookable (ทำได้จริง)**:
สถานะของ Recipe เทียบกับ Fridge: มีวัตถุดิบบังคับครบ มี Core Ingredient ที่ไม่ใช่โปรตีนครบทุกตัว และมีโปรตีนอย่างน้อยหนึ่งทางเลือก — เงื่อนไขเดียวที่ตัดสินว่าเมนูเข้ารอบไหม ไม่มีคะแนน/ไม่มีเกณฑ์บางส่วนอีกแล้ว
_Avoid_: Match Score, Coverage (แนวคิดเดิมที่ยอมให้เมนูขาดของเข้ารอบได้ — เลิกใช้เพราะผู้ใช้เคยเลือกไก่แล้วสุ่มได้เมนูกุ้ง)

**Candidate (เมนูเข้ารอบ)**:
Recipe ทุกตัวที่ Cookable จากของใน Fridge เป็นตัวตั้งต้นของการสุ่ม (uniform — ทุกตัวโอกาสเท่ากัน) และการสลับเมนู — ใน UI เรียก "ตัวเลือกสำรอง" แสดงไม่เกิน 6 ตัวแต่การสุ่มใช้ทั้งหมด

**Fridge (ตู้เย็น)**:
ชุด Ingredient ที่ผู้ใช้ระบุว่ามีอยู่ตอนนี้ เก็บถาวรในเครื่องของผู้ใช้เท่านั้น (ไม่ผูกกับ server) และคงอยู่ข้ามการปิดเปิดแอป
_Avoid_: selection, cart

**Tag (ป้ายกำกับ)**:
ป้ายอิสระของ Recipe เพื่อบรรยายลักษณะจาน เช่น เผ็ด, จานด่วน, เด็กกินได้ — **ไม่ใช่**การอ้างอิงวัตถุดิบ และไม่มีผลต่อ matching
_Avoid_: ใช้คำว่า tag กับวัตถุดิบบังคับ
