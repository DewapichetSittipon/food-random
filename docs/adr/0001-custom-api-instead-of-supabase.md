# ใช้ Custom API + DB แทน Supabase

สเปกเดิม (docs/FridgeChef Spec.dc.html) ระบุให้ sync คลังเมนูผ่าน Supabase แต่เราตัดสินใจเขียน API และฐานข้อมูลเอง เพราะงานฝั่ง backend ของ v1 มีขอบเขตแคบมาก (เสิร์ฟคลังเมนูแบบอ่านอย่างเดียวให้ฝั่งแอป + แก้ไขสูตรจากหลังบ้าน) จึงไม่คุ้มกับ lock-in และการตั้งค่า RLS/anon key ของ Supabase — แลกกับการที่เราต้อง host เองและสร้างช่องทางแก้ไขสูตรเอง

ฝั่งแอปยังคงเป็น Offline-First ตามสเปก: โหลดคลังเมนูลง IndexedDB แล้ว matching/random ทำงานในเครื่องทั้งหมด API ใช้เพื่อ sync คลังเมนูเท่านั้น
