# Sync คลังเมนูแบบ full snapshot แทน incremental updatedAt

สเปกเดิมให้ sync เฉพาะเมนูที่ `updatedAt` ใหม่กว่าที่มีในเครื่อง แต่วิธีนั้นไม่มีทางรับรู้ "การลบ" สูตรจากหลังบ้าน (ต้องเพิ่ม soft-delete + tombstone) ทั้งที่คลังเมนูมีขนาดหลักสิบถึงหลักร้อยรายการ เราจึงเลือกให้ `GET /catalog` คืนทั้งก้อน (ingredients + recipes) แล้วฝั่งแอปเขียนทับทั้งชุดใน transaction เดียวของ IndexedDB — การลบถูกจัดการโดยธรรมชาติ โค้ดทั้งสองฝั่งเรียบง่ายกว่ามาก

แอปอ่านจาก IndexedDB ก่อนเสมอ (เปิดติดทันทีแม้ออฟไลน์) แล้ว sync เบื้องหลังเมื่อออนไลน์ ใช้ updatedAt สูงสุดของทั้งคลังทำหน้าที่คล้าย ETag เพื่อเลี่ยงการเขียนซ้ำเมื่อไม่มีอะไรเปลี่ยน หากคลังโตระดับพันรายการค่อยกลับมาพิจารณา incremental ใหม่
